# 最小可行性产品 (MVP) 实施计划：同局域网分牌 - 欢迎页测试

我们将基于 GitHub Pages 搭建一个无需后端、通过 URL 传递状态的轻量级“实时同步”页面。

## 核心策略
*   **零成本同步**: 利用 **URL Hash (锚点)** 技术实现内容同步。
    *   *原理*: 页面上的文字内容会实时反映在 URL 中（例如 `.../index.html#欢迎各位`）。
    *   *优势*: 无需购买服务器或数据库，无需复杂的后端配置，完全免费且稳定。
*   **中国访问优化**:
    *   页面引用的所有脚本库（如生成二维码的库）将直接下载到本地（存入仓库），避免引用被墙的国外 CDN 链接，确保加载速度。
*   **GitHub Pages**: 使用 GitHub 自带的静态托管服务。

## 实施步骤

### 阶段一：环境准备 (Environment Setup)
1.  **克隆仓库**: 将 `youngzy607-cpu/Test_20251229` 克隆到本地当前目录。
2.  **清理/初始化**: 确保工作目录干净，准备好项目结构。

### 阶段二：功能开发 (Development)
1.  **创建核心页面 (`index.html`)**:
    *   显示一个巨大的“欢迎”文字区域（可编辑）。
    *   实现逻辑：当文字修改时，自动更新浏览器地址栏的 URL Hash。
2.  **集成二维码功能**:
    *   引入 `qrcode.js` (本地化)。
    *   添加“分享/生成二维码”按钮。
    *   点击后，根据当前带 Hash 的 URL 生成二维码。
3.  **样式美化**:
    *   简单适配手机端（响应式布局），确保文字和二维码在手机上清晰可见。

### 阶段三：部署与验证 (Deployment)
1.  **提交代码**: 将所有文件 Push 到 GitHub。
2.  **配置指引**: 指导你在 GitHub 仓库设置中开启 GitHub Pages。
3.  **最终测试**:
    *   验证链接是否生成。
    *   验证修改文字后，二维码扫出来的页面是否同步显示了新文字。
    *   验证 readme.md 是否更新。

## 为什么这个方案符合你的需求？
*   **实时同步**: 虽然没有后端，但通过二维码传递的是“带状态的链接”，实现了“我看什么，发给你，你就能看到什么”的效果。
*   **中国用户友好**: 纯静态页面，只要 GitHub 能访问，页面就能秒开（无额外 API 请求）。
*   **简单**: 纯前端实现，易于维护和理解。
